# Project requirements
- Nuxt 4. Documentation: [https://nuxt.com/llms.txt](https://nuxt.com/llms.txt).
- Nuxt UI 4. Documentation: [https://ui.nuxt.com/llms.txt](https://ui.nuxt.com/llms.txt).
- Tailwind CSS. Some documentation is available in the file tailwind-referential.md.
- For the icons, use Lucide icons. [https://icones.js.org/collection/lucide](https://icones.js.org/collection/lucide).
- Fo the documentation of the packages, refer to the correct versions of the packages as listed in the file package.json.

# Programming language
- JavaScript.
- Names of functions: camelCase.
- Names of variables: snake_case.
- Instead of using the `Date()` natitve JavaScript constructor, use the luxon library.

# Naming conventions
- Vue Components: PascalCase. For exemple: AppHeader.vue, AppFooter.vue.It makes components visually distinct from HTML elements in templates and is consistent with JavaScript class conventions.
- Nuxt pages: kebab-case. For clean, SEO-friendly URLs.
- Nuxt composables: camelCase.
- Other files and directories: kebab-case.
